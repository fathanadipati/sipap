1. Pendahuluan

AureliaBox adalah sebuah sistem manajemen paket yang dirancang khusus untuk mengelola alur pengiriman paket di apartemen premium. Sistem ini dikembangkan dengan tujuan memberikan solusi terpadu yang efisien dan user-friendly dalam menangani penerimaan, penyimpanan, dan pengambilan paket oleh penghuni.

Dikembangkan khusus untuk THE GRAND AURELIA RESIDENCE, aplikasi ini memadukan teknologi modern dengan desain antarmuka yang intuitif untuk meningkatkan pengalaman pengguna dan efisiensi operasional.

2. Latar Belakang & Tujuan

2.1 Latar Belakang

Setiap hari, apartemen premium menerima ratusan paket dari berbagai ekspedisi. Sistem penerimaan paket yang manual atau tidak terstruktur sering kali menyebabkan:

  • Kesalahan pencatatan data paket
  • Penghuni tidak mengetahui jika paketnya sudah tiba
  • Kesulitan dalam melacak status paket
  • Inefisiensi waktu penerimaan dan pengambilan paket
  • Kurangnya dokumentasi yang baik untuk keperluan audit

2.2 Tujuan Pengembangan

  • Menciptakan sistem manajemen paket yang terintegrasi dan efisien
  • Memberikan notifikasi real-time kepada penghuni saat paket tiba
  • Menyediakan dashboard monitoring untuk admin dan resepsionis
  • Mengotomatisasi proses penerimaan, penyimpanan, dan pengambilan paket
  • Menjaga keamanan data dan mengimplementasikan kontrol akses berbasis role
  • Memberikan user experience yang intuitif dan responsif

4.1 Alasan Pemilihan Teknologi

PHP Native: Lightweight, mudah di-deploy, tidak perlu dependency kompleks, cocok untuk XAMPP dan environment lokal.

MySQL: Reliable, widely used, support untuk relational database dengan foreign key dan integritas data.

Bootstrap 5: Modern UI components, responsive grid system, extensive documentation, dan community support yang besar.

Vanilla JavaScript: No dependencies, fast performance, native browser support untuk AJAX, tidak perlu build tools.

Apache: Standard web server, integrated dengan XAMPP, easy configuration, widely supported.

5. Rancangan Sistem

5.1 Arsitektur Aplikasi

AureliaBox menggunakan arsitektur 3-Tier (Presentation-Business Logic-Data) yang klasik dan proven:

  • Presentation Layer (Frontend): HTML, CSS, JavaScript untuk user interface yang responsif dan user-friendly
  • Business Logic Layer (Backend): PHP files yang menghandle business logic, validasi, dan proses bisnis
  • Data Access Layer (Database): MySQL untuk persistent data storage dengan relasi yang jelas

5.2 Workflow Paket Masuk

Berikut adalah workflow lengkap saat paket masuk ke sistem:

  1. Kurir datang dengan paket
  2. Resepsionis klik menu "Terima Paket Baru"
  3. Resepsionis mengisi form: nama pengirim, ekspedisi, jenis paket, deskripsi
  4. Resepsionis memilih unit/penghuni penerima paket dari dropdown
  5. Resepsionis memasukkan nomor loker tempat paket disimpan
  6. Resepsionis klik tombol "Simpan"
  7. Sistem auto-generate nomor paket unik
  8. Database update dengan status "Disimpan"
  9. Sistem otomatis mengirim notifikasi ke penghuni
  10. Penghuni menerima notifikasi real-time di dashboard
  11. Penghuni dapat melihat detail paket dan nomor lokernya

5.3 User Roles & Permissions

Role Permissions
Admin Manage users, manage residents, manage packages, view all data, access admin panel, generate reports
Resepsionis Receive packages, manage packages (edit/delete), view resident data, view notifications, print reports
Penghuni View own packages, view notifications, edit profile, view own data

8. Panduan Instalasi

8.1 Prasyarat (Requirements)

  • XAMPP (Apache + MySQL + PHP) versi terbaru
  • PHP 7.4 atau lebih tinggi
  • MySQL 5.7 atau lebih tinggi
  • Text Editor atau IDE (VS Code, PhpStorm, Sublime Text, dll)
  • Browser modern (Chrome, Firefox, Edge, Safari)
  • Koneksi internet untuk download XAMPP (jika belum ada)

8.2 Langkah Instalasi

1. Ekstrak file: Ekstrak folder "sipap" ke dalam folder "C:\xampp\htdocs\"

2. Start XAMPP: Buka XAMPP Control Panel dan start Apache dan MySQL

3. Import Database: Buka phpMyAdmin (http://localhost/phpmyadmin), buat database baru "sipap_db", lalu import file "database.sql"

4. Akses Aplikasi: Buka browser dan ketik: http://localhost/sipap

5. Login: Gunakan kredensial default untuk login sebagai admin

10. Keamanan Sistem

10.1 Implementasi Keamanan

Password Hashing

  • Menggunakan bcrypt untuk hashing password yang aman
  • Password tidak pernah disimpan dalam plain text
  • Salt otomatis pada setiap hash untuk security tambahan
  • Algorithm cost factor yang optimal untuk keamanan dan performa

SQL Injection Prevention

  • Menggunakan prepared statements dengan parameterized queries
  • Input validation di semua form sebelum dikirim ke database
  • Sanitasi data sebelum dimasukkan ke database
  • Escape special characters dengan proper escaping functions

XSS (Cross-Site Scripting) Prevention

  • Output encoding untuk semua user input yang ditampilkan
  • Htmlspecialchars() untuk mencegah script injection
  • Content Security Policy ready untuk HTTP headers
  • Input sanitasi untuk form fields

Session Management

  • Session timeout configuration untuk security
  • Unique session ID untuk setiap user login
  • Session data disimpan di server, bukan di client
  • Regenerate session ID setelah login berhasil
  • Proper session destroy pada logout

Access Control

  • RBAC (Role-Based Access Control) dengan 3 role yang jelas
  • Direct access protection untuk halaman admin
  • Permission check di setiap halaman yang restricted
  • 403 Forbidden page untuk akses terlarang yang dideteksi
  • Logging untuk attempted unauthorized access

Data Validation

  • Server-side validation di semua form submission
  • Input length checking untuk mencegah buffer overflow
  • Data type validation (email, phone, numeric, dll)
  • Required field validation untuk data penting
  • Custom validation rules sesuai business logic

10.2 Best Practices yang Diikuti

  • Never trust user input - selalu validasi dan sanitasi semua input dari user
  • Principle of Least Privilege - users hanya dapat akses fitur yang mereka butuhkan
  • Error handling - error tidak mengungkap detail teknis sistem
  • Logging - logging untuk audit trail dan troubleshooting
  • Regular backup - backup database secara berkala
  • HTTPS ready - saat hosting ke server, gunakan SSL certificate
  • Security headers - proper HTTP security headers implementation
  • Input/Output encoding - proper encoding untuk semua data
